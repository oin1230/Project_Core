
***** 회원정보
1. 회원가입시 이메일, 닉네임, 상세주소, 비밀번호, 지역을 입력받는다.
2. 회원은 이메일로 구분한다.

***** 행사목록
1. 행사목록은 크롤링으로 수집하는 정보이다.
2. 행사목록은 행사코드, 행사명, 행사날짜, 행사장소, 카테고리, 행사사진 으로 구성된다.
3. 행사목록은 행사코드로 구분한다.
4. 행사날짜는 년 월 일 시 분을 입력한다.
5. 카테고리는 콘서트, 축제로 구성된다.
****6. 행사사진은 이미지파일로 저장한다.
( 
	1. DB에 이미지파일로 저장하는게 나은지, 또는 다른이미지 폴더를 생성해 폴더에 관리하는게 나은지 질문필요
	2. DB에 이미지로 저장하게 된다면, 데이터타입은 어떻게?, 값은 어떻게? 
)

***** 셔틀목록
1. 셔틀목록은 셔틀코드, 행사코드, 출발지역, 총인원수, 모집시작일, 모집종료일, 가격, 승차일시, 진행상태로 구성된다.
2. 셔틀목록은 행사목록중에서 선택하여 등록할수 있디.
(
	1. 사용자가 등록하는것인지? 관리자가 등록하는것 인지? (
		사용자가 등록 할 경우 등록할 행사목록에서 등록할 출발지역이 등록이 안되어있을 경우
		선택해서 등록할 수 있게 할 것인지?
	)

3. 출발지역은 출발목록중에서 선택할 수 있다.(
	출발목록은 select태그의 옵션으로, 광역시 중에서 선택할 수 있게 한다.
)
4. 총인원수는 현재까지 모집된 인원수 이다.
5. 모집시작일, 모집종료일, 승차일시는 년월일시 로 입력한다.
6. 진행상태는 진행과 종료로 구성된다.
	( 

***** 신청목록
1. 신청목록은 회원이 셔틀목록에서 선택하여 신청할 때 생성된다,
2. 신청목록은 신청코드, 셔틀목록의 셔틀코드, 회원정보의 이메일, 신청원인수로 구성된다.
3. 신청인원수는 회원이 신청할 때 입력해야 한다.

**** 4. (선택한 좌석을 입력하는게 맞는지?
	그렇다면, 신청인원수가 다수여서, 좌석을 여러개 선택할 경우는 어떻게 처리해야 할지?
)

***** 좌석
1. 좌석은 좌석코드, 신청목록의 신청코드, 셔틀목록의 셔틀코드, 버스 카테고리, 좌석상태으로 구성된다.
2. 좌석은 셔틀목록에서 선택할 수 있는 자리를 구분하는데 사용된다
3. 버스 카테고리는 15인승 25인승 45인승등, 버스를 구분하는데 사용된다.
4. 좌석은 결제할 때 선택한 좌석번호를 나타낸다.

***** 결제
1. 결제는 결제코드, 신청목록의 신청코드, 결제수단, 결제일자, 결제금액, 선택좌석 으로 구성된다
2. 셔틀목록의 총인원수가 일정수치가 넘으면, 신청목록에 결제탭이 활성화 되어, 결제를 할 수 있고, 좌석을 선택해야한다.

***** 게시판
1. 게시판은 게시글 코드, 회원정보의 이메일, 카테고리, 제목, 내용, 작성시간으로 구성된다
2. 로그인한 회원만 게시글을 작성할 수 있다.
3. 카테고리는 잡담, 정보, 모집등 글머리를 정할 수 있다.
4. 작성시간은 년 월 일 시로 입력된다
